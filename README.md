# RuleKit برای my-data-quality

RuleKit مجموعهای از توابع ساده در R برای تعریف و ارزیابی قوانین کیفیت داده است.

- خروجی قوانین: TRUE = نقض، FALSE = عدم نقض، NA = خارج از دامنه ارزیابی
- سیاست گیت:
  - اگر قانون شامل `val` باشد، فقط زمانی ارزیابی میشود که مقدار آن سطر برای ستون هدف موجود باشد؛ در غیر این صورت `NA`.
  - اگر قانون شامل `val` نباشد (قوانین چندستونه)، فقط زمانی ارزیابی میشود که تمام ستونهای مرجع در آن سطر مقدار داشته باشند؛ در غیر این صورت `NA`.

## فایلها
- `rulekit.R`: هستهی ارزیابی قوانین
- `example_usage.R`: مثال اجرا
- `README.md`: این راهنما

## شروع سریع

1) فایلها را در ریشه مخزن قرار دهید.
2) در R اجرا کنید:

```r
source("rulekit.R")
source("example_usage.R")
```

## تعریف قانون

- قانون تکستونه (با استفاده از `val`):
  - مثال: `val >= 0 & val <= 100` (محدوده مجاز)
  - مثال: `!is.na(val)` (نباید مقدار مفقود باشد)

- قانون چندستونه (بدون `val`):
  - مثال: `A <= B`
  - مثال: `A + B == C`

> نکته: از عملگرهای برداری `&` و `|` بهجای `&&` و `||` استفاده کنید.

## قالب جدول قوانین

یک `data.frame` با ستونهای زیر:
- `name` (اختیاری ولی توصیهشده): نام یکتای قانون
- `variable`: نام ستون هدف برای قوانینی که `val` دارند (برای قوانین چندستونه `NA` بگذارید)
- `rule`: متن قانون (عبارت R)
- `severity` (اختیاری): مثل `info`, `warn`, `error`

مثال:
```r
rules <- data.frame(
  name = c("A_range", "B_required"),
  variable = c("A", "B"),
  rule = c("val >= 0 & val <= 100", "!is.na(val)"),
  severity = c("warn", "error")
)
```

## خروجیها

- `out$results`: نتایج سطری هر قانون + ستون `violation_count` برای شمارش نقضها در هر ردیف
- `out$rule_summary`: خلاصه تعداد نقض/عدمنقض/NA برای هر قانون
- `out$diagnostics`: هشدارهای نوشتاری و وابستگی ستونها برای هر قانون

## نکات
- اگر عبارت قانون بهدرستی پارس نشود یا خطای ارزیابی داشته باشد، نتیجه آن قانون `NA` میشود و پیام خطا در diagnostics بازتاب میشود.
- برای قوانین بدون `val`، اگر هر ستون مرجع در یک سطر `NA` باشد، نتیجه همان سطر برای آن قانون `NA` میشود.
